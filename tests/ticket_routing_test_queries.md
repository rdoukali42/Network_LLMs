# Ticket Routing Test Queries and Expected Assignments

This document contains all test queries used to validate the HR Agent ticket routing system, along with expected assignments and additional test examples.

## Overview

The HR Agent system routes tickets to appropriate employees based on:
- Employee expertise and skills
- Department alignment
- Current availability and workload
- AI-powered matching algorithms

For document-based queries, the system uses the Data Guardian to search local company documents instead of assigning to employees.

## Test Validation Criteria

### Employee Assignment Success Criteria
1. **Skill Match**: Assigned employee has relevant expertise for the ticket type
2. **Availability**: Employee is currently available and not overloaded
3. **Department Alignment**: Employee's role aligns with ticket requirements
4. **AI Confidence**: Matching algorithm confidence score above threshold (>0.7)
5. **Assignment Reasoning**: Clear explanation for why employee was selected

### Document Query Success Criteria
1. **Content Relevance**: Response contains information directly related to the query
2. **Source Identification**: Correct document(s) identified and searched
3. **No HR Assignment**: System bypasses employee assignment for document queries
4. **Answer Completeness**: Response addresses the key aspects of the question
5. **Processing Efficiency**: Quick retrieval without unnecessary employee routing

### System Performance Metrics
- **Success Rate**: Percentage of tickets correctly routed (Target: >90%)
- **Processing Time**: Average time to route tickets (Target: <10 seconds)
- **Employee Utilization**: Balanced distribution across available employees
- **Document Hit Rate**: Percentage of document queries answered successfully
- **User Satisfaction**: Quality and appropriateness of assignments

## Notes for Test Maintenance

1. **Employee Database Updates**: When new employees are added, update expected assignments accordingly
2. **Skill Evolution**: As employees gain new skills, test criteria may need adjustment  
3. **Document Updates**: Changes to company policy documents may affect document query responses
4. **AI Model Updates**: LLM model changes might affect matching logic and reasoning
5. **Performance Benchmarks**: Regularly review and update target performance metrics

This comprehensive test suite ensures the HR Agent system correctly routes tickets to appropriate employees or finds answers in company documents, maintaining high accuracy and user satisfaction.
